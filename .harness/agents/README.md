# Harness Agents

역할별 프롬프트 파일 모음. 어떤 AI 툴에서든 복붙해서 쓸 수 있는 "전문가 프롬프트" 집합입니다.

## 왜 역할을 나누나요?

AI에게 "사이트 좀 고쳐줘"라고 하면 결과가 일관되지 않아요. 카피·디자인·코드·검수를 한꺼번에 하면 품질이 떨어져요.

각 역할마다 **담당하는 일 / 안 하는 일 / 작업 순서**를 명확히 정의해두면 결과가 훨씬 일관되고 좋아집니다. 요즘 유행하는 "하네스 엔지니어링" 패턴이에요.

## 어떤 역할들이 있나요?

| 파일 | 역할 | 언제 씀 |
|---|---|---|
| [planner.md](./planner.md) | 계획가 | 복잡한 요청을 작은 작업으로 쪼갤 때 |
| [copywriter.md](./copywriter.md) | 카피라이터 | 문구, 제목, 버튼 텍스트 쓸 때 |
| [designer.md](./designer.md) | 디자이너 | 색상, 폰트, 레이아웃 정할 때 |
| [builder.md](./builder.md) | 빌더 | 새 컴포넌트/섹션/페이지 추가할 때 |
| [reviewer.md](./reviewer.md) | 리뷰어 | 접근성/SEO/모바일 점검할 때 |
| [debugger.md](./debugger.md) | 디버거 | 에러가 날 때 원인 찾기 |

## 어떻게 쓰나요?

### Claude Code (가장 간단)

```
.harness/agents/copywriter.md 역할로 Hero 섹션 카피 다시 써줘
```

Claude가 파일을 읽고 그 역할대로 작동해요. 서브에이전트 기능이 있으면 자동으로 활용해요.

### Codex

```
@.harness/agents/copywriter.md

Hero 섹션 카피 다시 써줘
```

`@`로 파일을 참조하면 Codex가 그 내용을 컨텍스트로 삼아요.

### Antigravity / Claude Desktop

파일 내용을 복사해서 채팅 시작할 때 붙여넣기:

```
[copywriter.md 내용 전체]

---

이제 Hero 섹션 카피 다시 써줘.
```

### Cursor / Windsurf

`.cursorrules` 대신 개별 역할을 그때그때 지정할 수 있어요. 위 Codex 방식과 동일.

## 권장 작업 순서

새 기능 추가할 때는 이 순서로:

```
planner  →  designer  →  copywriter  →  builder  →  reviewer
  (기획)       (디자인)       (카피)         (구현)      (검수)
```

버그 고칠 때:

```
debugger  →  builder  →  reviewer
 (진단)       (수정)       (검수)
```

## 여러 역할 한 번에?

**비추천.** 한 AI가 한 번에 한 역할씩 맡는 게 결과 품질이 훨씬 좋아요.
단, 시간이 없을 땐 `planner`가 쪼갠 뒤 각 작업을 순서대로 한 채팅에서 처리해도 OK.

## 커스터마이징

이 파일들은 **시작점**이에요. 프로젝트에 맞게 자유롭게 수정하세요:
- 브랜드 고유 용어 추가
- 피해야 할 표현 추가
- 프로젝트 특화 제약 추가 (예: "한국어 존댓말 사용", "Lucide 아이콘만")

각 파일 상단의 YAML 블록은 역할 메타데이터예요. 멀티 에이전트 자동화를 지원하는 툴(예: MoAI)은 이걸 파싱해서 자동 체이닝해요. 없어도 일반 AI에선 그냥 프롬프트처럼 작동하니 걱정 마세요.
